Excel Interiors' customer management application is an end to end React application I developed and designed using:
- React
- Redux
- Node.js
- PostgreSQL
- Auth0
- SASS
- Express
- Stripe

The sign in page features a high resolution background image to cover the viewport and once the sign in button is clicked the user signs in using Auth0.

When the user signs in, a function runs to determine if the user is an admin. If true, the customers page renders.

Here the admin is able to search through all the customers in the database using the search bar, that is neither case sensitive or order sensitive, powered by a SQL query.

The create new customer page is rendered when the admin clicks on the plus sign in the lower right hand corner.

When the save button is clicked a post request is made to the database which adds a new customer to the appropriate table.

To render the invoices page a get request is made to grab all invoices and a SQL query joining two tables together allows for the appropriate information to render.

Once a customer signs in, a function is ran to grab their invoices by their ID and render them on the front end.

With the help of a function, some inline styling and a ternary, the customers navigational option dissapears from view, so their not granted access to all the customers in the database.

Once the customer clicks on thier invoice a get request is made to grab all appropriate invoice information.

The customer can now click "Make A Payment" to pay using the Stripe API.
